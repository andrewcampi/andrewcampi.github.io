Slow Loris Attack

DISCLAIMER: Denial of service attacks are illegal. This document is strictly informational and is in no regard promoting any use of denial of service attacks.

Overview: 

A Slow Loris Attack is a denial of service attack that has a very interesting and unique attack strategy.
The objective of any denial of service attack is to make a website, server, computer, or other service unoperational. This is achieved by giving it the maximum possible bandwidth or by requesting the maximum amount of web pages from a website such that it breaks.
A Slow Loris Attack achieves the same goal as a brute force attack except in a completely opposite approach. It is a protocol attack, which means that it can be done with very minimal bandwidth and computing power.  

Communicating With a Web Server:
In order to display a webpage, a client’s web browser will have little individual conversations with the web server. For example, to display the homepage of the website, the client will send a get request for index.html and maybe another get request for some jpg files to display images. Most of the time, an http request is simply text. The get request follows a basic format, containing the words “http get”, some information about the client such as the name of the web browser, and some other data. It always ends with two character return line feeds (two new empty lines) to signal the end of the get request. The slow loris attack exploits the get request’s expectation of these two character return lines. The web server will wait until it receives the end of the get request, but if the two character return lines are never sent, it will wait for a long time. These get requests are so slow that it breaks the web server, hence the name “slow” loris.
Attacker vs Web Server:
If the get request is not finished in a certain amount of time, the web server will time out the connection, terminating it. However, if another single byte of information is sent from the client to the server just before it terminates the connection, the time until which the server will drop the connection is reset. In doing so, the server will hold a connection with a client even though the client is barely sending any information at all. An attacker can exploit this by establishing hundreds of slow loris connections with a single server. Even if the server does not break, the server will eventually waste hundreds of connection opportunities to clients due to the connections with the one attacker. It is very difficult for the server’s firewall to recognize this since they are super slow, yet valid, http requests.
Apache Servers:
About 50% of active websites are hosted on Apache servers. These are the servers that are most vulnerable to slow loris attacks because Apache servers create a new thread to server every concurrent connection. When a new http request is received by the Apache server, it sets up a new thread that handles the request, and removes it when it is finished serving the request. This is a problem due to Apache servers having a connection limit. If, for example, a certain Apache server has a connection limit of 100, then a slow loris attacker can consume the entire connection limit of a server simply by using about 200 bytes of bandwidth per minute.
