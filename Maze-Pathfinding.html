Maze Pathfinding

Objective:

To use the stack abstract data type (ADT), recursion, and the A* search algorithm to solve mazes.

Overview:
Assume that a maze is a rectangular array of squares, some of which are blocked, others are empty (call them a passage). All squares at the boundary of the maze that are not blocked are possible exits, if there is a way leading to it from the entrance point. The entrance point may be any passage square of the maze, not necessarily one on the boundary. A path from the entrance to an exit can consist of only passage squares that are next to each other. There may be many exists for a given maze and the entrance point. A mouse, or any other creature with pathfinding intelligence, chooses an entrance among the squares that are passages and it will try to find all possible ways leading to an exit from the given entrance. 

In the given maze, the asterisks represent the squares that are blocked and the spaces represent the possible passage. Starting at location (5,6), one will find the three possible paths to the exits through a different path. The squares on the path are represented by exclamation points. There are four copies of the given maze. The upper left maze is the original, and the other mazes show three different paths from the entrance to the three exits.

Movement Rules:
The basic rules of moving around the maze are as follows:
1. Then entrance location and the exit locations can be anywhere in the maze. The example in the above maze limits exit locations on the edges.
2. The mouse starts at the location within the maze specified as the entrance point.
3. The mouse can move through any open space (possible passage) toward the exits which are specified by a pair of coordinates. The squares at the exits have the possible passages themselves.
4. In this project, the mouse can only move in four directions: up, down, left, and right. The mouse cannot move diagonally, although permitting it to do so would only double the complexity of the algorithm.
5. The mouse should be able to find all possible paths, not just one.
How It Works:
Starting with the current mouse location (e.g. the entrance point), the mouse will look for a possible move in all four directions if the current square is not an exit. If one foteh neighboring squares is a possible passage and it has not been visited before, the mouse will move to that square by marking the square as visited as a part of a path. The algorithm uses recursion, a stack, or A* until the current location is the exit location. 
The Program’s Process:
The program randomly generates a maze or reads a given maze from a text file and displays it on the screen. This should depend on the user choice. The user is then able to specify a particular entrance point and a particular exit point after viewing the maze. The program then takes the pair of input and exit coordinates and invokes the necessary methods in various objects to find a path between the entrance point and the exit point. The program then displays all the correct paths to the user on the screen. 
Recursion vs Stack:
The algorithm to find a maze path using recursion is very similar to that of using a stack. The only difference is that in recursion, we recursively call a function instead of using a while loop until the maze is solved. In this algorithm, a local deep copy of the current maze state needs to be made before exploring the next direction.

A* Algorithm:

Using a stack or recursion to solve the maze is a brute force method of pathfinding and does not make for a very intelligent algorithm. Instead of searching every direction, A* creates a vector from the current position to the exit, and it chooses the possible path that is closest to the direction of the vector.

Dead ends cause trouble for this algorithm, since the direction of the vector was misleading. This is when the algorithm uses backtracking, tracing its steps back to where it made its most recent path decision. It then chooses the next best path option, and continues the A* algorithm for each position in the maze until it is solved or it needs to backtrack again. This algorithm not only is more time efficient than a brute force algorithm, but it also has a greater likelihood of finding the shortest path from the entrance to the exit.
