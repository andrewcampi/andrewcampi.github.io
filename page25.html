Tor Hidden Services

Overview:

The vast majority of what happens on Tor is completely legitimate: people browsing the web anonymously. However, some people use Tor to conduct illegal services, in Tor hidden services (a.k.a. the dark web). Some argue that all of Tor should be illegal due to this criminal activity.

The provided graphic visualizes a general Tor cloud. There are many people going in, many encrypted layers and messages being bounced around in the cloud, and many connections to servers coming out.

The issue, from a security standpoint, is that if there is someone data sniffing at point A and point B, the client's activity can be understood by correlating the traffic between the two points.

A hidden service eliminates the ability to data sniff at point B by moving inside the cloud. 

The hidden services circuit never leaves the circuit, so no traffic correlation can be done between two points.  
The Protocol:
When communicating with a hidden service, neither the client nor the server know who each other are, but can still communicate. 

When the server is turned online, it picks three onion routers at random and names them as introduction points. These onion routers can still perform other tasks while the hidden service uses them as an introduction point. The circuit between the server and the introduction point is a full Tor circuit with at least three hops.

With this circuit, the introduction points know that they are going to be introducing clients to the server, but they do not know who the client is.

When a server establishes its introduction points, it also creates a hidden service descriptor. The descriptor includes the service public key, for authentication purposes, and the ip addresses of the introduction points. The server will then publish the descriptor to a distributed hash table, meaning that every onion router on Tor holds some part of the information on every hidden service. Therefore, if a client tries to look up a hidden service, the router responsible for it will give the client the descriptor, so that the client can establish a connection with one of the hidden service’s introduction points.

The key for the hash table is the onion address, which is derived from the public key of the hidden service. It gets the ip addresses out of the distributed hash table. The point of the hash table is that the onion address is not publicized on the global Tor network. To find the onion address, a client needs to find someone who personally knows it. This made it very difficult to calculate how many hidden services there are on the Tor network. 

 Establishing The Connection To An Introduction Point:
Since the client has the onion address, the client can request the descriptor off the hash table. The client is returned the descriptor, which contains the three ip addresses of the introduction points to the server.
The client then chooses a rendezvous point at random and creates a full Tor circuit to that rendezvous point. This Tor circuit is made up of two nodes and the rendezvous point as the third node, which will connect to an introduction point on the clients behalf.

The client then sends a password to the rendezvous point, who sends it to the introduction point. if it is a successful connection, the introduction point will send it back to the rendezvous point, who sends it back to the client. This verifies that one one else has gotten involved with the conversation. The introduction point then sends the password to the server. At this point, the server can respond or do nothing (depending on if the password is correct). 

If the server wants to make a connection, the server establishes another full Tor circuit with the rendezvous point. 

The server then sends a “rendezvous please” to the rendezvous point, which contains the password that the server received from the introduction point. The rendezvous point then compares the two passwords, and if they match, establishes a connection between the client and the server through itself. There are now at least six nodes between the client and the server. 
Applications:
Some servers don’t need anonymity and this many hops is too slow. They want to protect their clients but they do not care if people know who the server is. Facebook recently worked with Tor to introduce a “single onion.” It follows a similar protocol as a hidden service, except that the server establishes a connection with the rendezvous point directly instead of making a full Tor circuit.
